### I design systems before I write code.

</div>

<br/>

I'm a **backend-focused engineer** who builds scalable, production-grade systems and AI-powered products. My work sits at the intersection of **distributed architecture**, **AI/LLM infrastructure**, and **real-world product delivery**.

I don't just implement features — I think about **how systems fail**, **where they break under load**, and **what it takes to make them reliable at scale**. Whether it's a RAG pipeline, a microservice mesh, or a full backend from scratch — I care about the fundamentals that make software production-worthy.

Currently building **[FitBuddy](https://github.com/Shrikrushn07)** and **[Saksham AI](https://github.com/Shrikrushn07)** — and deepening expertise in Kubernetes, distributed systems, and LLM orchestration.

## Engineering Beyond Code

<br/>

Most developers write code. I think about the **system around the code** — how it's deployed, how it scales, how it fails gracefully, and how the next engineer can understand it.

My DevOps practice is built on these principles:

- **Containerisation** — every application I build runs in Docker, reproducible across environments
- **Orchestration** — learning Kubernetes for service scaling, rolling deployments, and resilience
- **CI/CD** — automated pipelines via GitHub Actions so shipping is boring and reliable
- **Microservices** — decomposing monoliths into independently deployable, loosely coupled services
- **Observability** — understanding that a system you can't monitor is a system you don't understand

> *"Production is the real test. Everything else is rehearsal."*

name: Generate Snake

on:
  schedule:
    - cron: "0 0 * * *"
  workflow_dispatch:

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: Platane/snk@v3
        with:
          github_user_name: Shrikrushn07
          outputs: |
            dist/github-snake.svg
            dist/github-snake-dark.svg?palette=github-dark

          target_branch: output
